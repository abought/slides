Concept 1

Data takes time to transfer

Each file gets downloaded separately

Every request incurs some time waiting

Waiting makes things slow

Practice 1

Asset concatenation

// Put two files of code in a single file
// Fewer requests, less waiting!
function myCode() {
  return 12;
}

function yourCode() {
  return 24;
}
You may see these files called "app.js" (your code) and "vendor.js" (shared libraries)

Concept 2: Big files take time

Make them smaller

Practice 2

Uglification / minification

Before
/**
* A comment meant to supply documentation for developers
*/
function myFunction(param1, param2, unusedParam) {
  if (param1 > param2) {
    return 12;
  }
  return param1 + param2;
}
After:
function myFunction(n,t){return n>t?12:n+t}

Concept 3

Library and package management

Practice 3a

Keep dependencies separate

Make it clear what third party code you depend on
But keep it separate- easier to upgrade and track
(Usually these will be in bower.json and package.json)

Practice 3b

Keep code isolated (with modules)

In JavaScript, everything is global

Concept 4

Tomorrow's code, yesterday

Practice 4 (optional)

Transpilers and polyfills

Programming languages sometimes get new features. But old browsers can't run them.
Transpilers rewrite new syntax into old syntax. Polyfills let you use new features that aren't actually there.

Concept 5

The code you write vs the code you run

Practice 5

Source maps

The browser may see different code. This makes debugging hard.

Source maps connect the code you wrote to the error message you see in the browser. Use them!
